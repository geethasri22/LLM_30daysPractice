# 🤖 What is Prompt Engineering?

Prompt Engineering is the practice of designing clear, structured, and context-rich instructions that guide a Large Language Model to produce accurate, relevant, and useful responses.

A prompt is more than just a question—it can define the model's role, provide context, specify constraints, and describe the desired output format.

---

# Why Prompt Engineering Matters

A well-crafted prompt can significantly improve the quality of an LLM's response without changing the underlying model.

Good prompts help:

* Improve accuracy
* Reduce ambiguity
* Control output format
* Encourage reasoning
* Produce consistent results

---

# Anatomy of a Good Prompt

A strong prompt typically includes:

* **Role:** Who the model should act as.
* **Task:** What it should do.
* **Context:** Background information.
* **Constraints:** Rules or limitations.
* **Output Format:** The expected structure of the answer.

### Example

❌ Poor Prompt

```
Explain AI.
```

✅ Better Prompt

```
You are an AI professor.

Explain Artificial Intelligence to a 10-year-old child using simple language and one real-life example.

Keep the explanation under 150 words.
```

---

# Types of Prompting

## 1. Zero-shot Prompting

The model receives only the task.

```
Translate this sentence into French:

I love learning AI.
```

---

## 2. One-shot Prompting

The model is given one example.

```
English: Hello
French: Bonjour

English: Thank you
French:
```

---

## 3. Few-shot Prompting

The model receives multiple examples before solving the task.

```
Positive → I love this movie.

Negative → I hate this movie.

Positive → The food was amazing.

Negative →
```
